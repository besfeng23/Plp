# PLP Luxury UI Standard

This document is the operating standard for moving Pueblo La Perla Boracay toward an Aman, Raffles, and Four Seasons level digital experience.

## Positioning

Pueblo La Perla is not a transaction-first booking site. It is a private hillside retreat in Boracay.

The website must feel:

- private
- quiet
- elevated
- discreet
- warm
- concierge-led
- visually atmospheric

It must never feel:

- fintech-first
- promotional
- cheap
- noisy
- overexplained
- unfinished
- generic

## Guest promise

The site should communicate one simple promise:

> A quieter way to stay in Boracay, handled with private resort discretion.

Every page should answer:

1. What is this place?
2. Why does it feel special?
3. What can I stay in?
4. What can the team arrange?
5. How do I begin a private reservation?

## Visual rules

### Photography

No visible grey placeholders are allowed on public-facing pages.

Acceptable visuals:

- real PLP resort photography
- aerial hillside villa imagery
- villa pool imagery
- interior room photography
- Boracay atmosphere imagery
- water, wellness, arrival, and private dining visuals
- cinematic fallback treatments using approved PLP images only

Unacceptable visuals:

- generic stock hotel rooms
- low-resolution screenshots
- stretched images
- grey placeholder blocks
- labels such as `[ Villa Image ]` on the live site

### Image behavior

Hero visuals should feel immersive and cinematic.

Below-the-fold visuals should be lazy-loaded.

Every content image should have meaningful alt text.

### Color palette

Primary colors:

- Ink: `#1A1A1A`
- Cream: `#FAFAF7`
- Soft stone: `#F3EFE6`
- Resort gold: `#7A6A3A`
- Deep black: `#050505`

Do not use bright fintech blues, greens, or payment-brand colors in public resort pages.

### Typography

Use only two font families:

- Cormorant Garamond for editorial headings
- Inter for functional text and navigation

Headings should breathe. Body copy should be short and readable.

### Spacing

Luxury spacing is not empty by accident. Large white space must frame important content or imagery.

Avoid accidental dead zones caused by missing media.

## Copy rules

Use:

- Reserve your stay
- Private reservation
- Preferred dates
- Stay summary
- Concierge review
- Arrival details
- Final confirmation follows review
- The team will arrange
- A reservation hold may be requested

Avoid public-facing language like:

- Pay now
- Powered by Xendit
- Deposit-first
- Checkout
- Secure payment
- Online installment
- Transaction

Payment language belongs only inside the reservation completion flow, never on the homepage.

## Navigation rules

Public navigation should stay minimal:

- Overview
- Accommodation
- Experiences
- Exclusive Offer
- Getting Here
- Reserve

Mobile navigation must prioritize:

- Reserve
- Email
- Concierge

## Booking rules

The booking experience should feel like a private concierge request.

It should include:

- room selection
- date selection
- guest details
- live availability check
- stay estimate
- clear final confirmation explanation
- direct concierge fallback

It should avoid sounding like a payment gateway.

## Trust rules

Trust should be built through:

- clear policies
- direct contact
- social proof
- real imagery
- arrival guidance
- transparent rates
- calm reservation explanations

Trust should not be built through cheap payment badges.

## Accessibility rules

All public pages must preserve:

- readable contrast
- visible focus states
- keyboard navigation
- meaningful alt text
- comfortable mobile tap targets
- body text that can be read outdoors on a phone

## Standard of judgment

A page passes only if it feels:

> quiet enough for Aman, polished enough for Raffles, and operationally clear enough for Four Seasons.
